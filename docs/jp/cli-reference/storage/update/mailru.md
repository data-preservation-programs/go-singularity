# Mail.ru Cloud

{% code fullWidth="true" %}
```
NAME:
   singularity storage update mailru - Mail.ru Cloud

USAGE:
   singularity storage update mailru [command options] <name|id>

DESCRIPTION:
   --user
      ユーザー名（通常はメールアドレス）。

   --pass
      パスワード。
      
      これはアプリのパスワードである必要があります。通常のパスワードではrcloneは動作しません。
      アプリパスワードの作成方法については、ドキュメントの構成セクションを参照してください。
      

   --speedup-enable
      同じデータハッシュを持つ別のファイルがある場合、完全なアップロードをスキップします。
      
      この機能は「スピードアップ」または「ハッシュを使用して入れる」と呼ばれます。
      一般に利用可能なファイル（人気のある本、ビデオクリップ、オーディオクリップなど）の場合、
      ハッシュでファイルを検索しています。ソースファイルがユニークまたは暗号化されている場合、
      無意味で効果がありません。
      rcloneは事前にコンテンツのハッシュを計算し、完全なアップロードが必要かどうかを決定するために
      ローカルのメモリとディスクスペースが必要になる場合があります。
      また、rcloneが事前にファイルサイズを知らない場合（ストリーミングや部分的なアップロードの場合など）
      この最適化を試みることはありません。

      例：
         | true  | 有効にする
         | false | 無効にする

   --speedup-file-patterns
      スピードアップ（ハッシュによる入れる）の対象となるファイル名のパターンのコンマ区切りリスト。
      
      パターンは大文字小文字を区別せず、'*'または'?'のメタ文字を含むことができます。

      例：
         | <unset>                 | リストが空の場合、スピードアップ（ハッシュによる入れる）は完全に無効になります。
         | *                       | すべてのファイルがスピードアップに試されます。
         | *.mkv,*.avi,*.mp4,*.mp3 | 一般的なオーディオ/ビデオファイルのみがスピードアップに試されます。
         | *.zip,*.gz,*.rar,*.pdf  | 一般的なアーカイブまたはPDFのみがスピードアップに試されます。

   --speedup-max-disk
      大容量ファイルのスピードアップ（ハッシュによる入れる）を無効にする場合のオプション。
      
      事前のハッシュ計算により、RAMまたはディスクスペースが枯渇する可能性があるためです。

      例：
         | 0  | スピードアップ（ハッシュによる入れる）を完全に無効にします。
         | 1G | 1GBより大きいファイルは直接アップロードされます。
         | 3G | ローカルディスクの空き容量が3GB以下の場合、このオプションを選択してください。

   --speedup-max-memory
      指定されたサイズよりも大きいファイルは常にディスク上でハッシュ化されます。

      例：
         | 0    | 事前のハッシュ計算は常に一時的なディスク領域で行われます。
         | 32M  | 事前のハッシュ計算に32MB以上のRAMを割り当てないでください。
         | 256M | ハッシュの計算に最大256MBのRAMが利用可能です。

   --check-hash
      ファイルのチェックサムが不一致または無効な場合に、コピーがどのように動作するかを指定します。

      例：
         | true  | エラーで失敗する
         | false | 無視して続行する

   --user-agent
      クライアント内部で使用されるHTTPユーザーエージェント。

      デフォルトは「rclone/VERSION」またはコマンドラインで指定された「--user-agent」です。

   --quirks
      内部メンテナンスフラグのコンマ区切りリスト。
      
      このオプションは通常のユーザーには使用しないでください。これはバックエンドの問題を遠隔トラブルシューティング
      するためだけに意図されています。フラグの厳密な意味は文書化されておらず、リリース間で永続することは保証されていません。

      サポートする内部メンテナンスフラグ：atomicmkdir binlist unknowndirs

   --encoding
      バックエンドのエンコーディング。

      詳細については、[概要のエンコーディングセクション](/overview/#encoding)を参照してください。


OPTIONS:
   --help, -h        ヘルプを表示
   --pass value      パスワード [$PASS]
   --speedup-enable  同じデータハッシュを持つ別のファイルがある場合、完全なアップロードをスキップします。 (デフォルト: true) [$SPEEDUP_ENABLE]
   --user value      ユーザー名（通常はメールアドレス） [$USER]

   Advanced

   --check-hash                   ファイルのチェックサムが不一致または無効な場合に、コピーがどのように動作するかを指定します。 (デフォルト: true) [$CHECK_HASH]
   --encoding value               バックエンドのエンコーディング。 (デフォルト: "Slash,LtGt,DoubleQuote,Colon,Question,Asterisk,Pipe,BackSlash,Del,Ctl,InvalidUtf8,Dot") [$ENCODING]
   --quirks value                 内部メンテナンスフラグのコンマ区切りリスト。 [$QUIRKS]
   --speedup-file-patterns value  スピードアップ（ハッシュによる入れる）の対象となるファイル名のパターンのコンマ区切りリスト。 (デフォルト: "*.mkv,*.avi,*.mp4,*.mp3,*.zip,*.gz,*.rar,*.pdf") [$SPEEDUP_FILE_PATTERNS]
   --speedup-max-disk value       大容量ファイルのスピードアップ（ハッシュによる入れる）を無効にする場合のオプション。 (デフォルト: "3Gi") [$SPEEDUP_MAX_DISK]
   --speedup-max-memory value     指定されたサイズよりも大きいファイルは常にディスク上でハッシュ化されます。 (デフォルト: "32Mi") [$SPEEDUP_MAX_MEMORY]
   --user-agent value             クライアント内部で使用されるHTTPユーザーエージェント。 [$USER_AGENT]

```
